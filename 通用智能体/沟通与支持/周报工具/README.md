# 周报工具

## 简介

周报工具是一个工作周报整理助手，能将用户散乱的周工作内容进行结构化梳理，生成简洁完整、重点清晰的中等篇幅周报，并按职场邮件规范排版，最终通过邮件发送给主管。

## 功能特性

### 1. 四大核心模块
自动从用户输入中提取以下四个模块（无对应信息可省略）：

| 模块 | 处理要点 |
|------|---------|
| 核心工作事项 | 区分完成 / 推进中 |
| 工作成果 / 进展 | 优先量化，无量则描述完成度 |
| 问题 / 待协调事项 | 如有则列出，无则标"暂无" |
| 下周初步计划 | 无明确信息时使用预设表述 |

### 2. 内容处理规则
- 整合散乱表述，保留关键、剔除口语冗余
- 正式但不生硬
- 同类工作合并不重复

### 3. 排版输出
- 模块加清晰标题
- 分点呈现，要点简洁
- 按职场邮件规范排版周报内容
- 邮件内容使用 HTML 格式

### 4. 邮件发送
- 仅在用户提供主管邮件地址后执行发送
- 不擅自发送邮件
- 邮件主题："工作周报 - [姓名] - [日期周期]"
- 支持抄送（CC）参数

### 5. 信息原则
- 严格基于用户输入整理
- 不杜撰、不额外添加信息
- 内容过简时只做通顺化处理，不随意扩充

## 技术架构

| 工具名称 | 功能说明 |
|---------|---------|
| `send_email` | 发送 HTML 格式邮件（支持 CC） |

## 工作流程

```
用户输入散乱的工作内容
  ↓
拆分到四大模块
  ├─ 核心工作事项（完成 / 推进中）
  ├─ 工作成果 / 进展（量化或完成度）
  ├─ 问题 / 待协调事项（如有）
  └─ 下周初步计划（如无则用预设表述）
        ↓
整合同类、精简冗余、正式化处理
        ↓
按 HTML 邮件规范排版
        ↓
询问主管邮箱地址
        ↓
使用 send_email 发送
```

## 使用示例

**示例 1：仅整理周报内容（不发送）**

用户输入：
```
这周主要做了A项目的数据分析，写了个报告。还开了个会，讨论B功能的方案。帮小张处理了一个线上bug。下周打算继续优化B方案，然后调研C技术。A项目数据源有问题，需要小李帮忙。
```

Agent 输出结构化周报：

```
一、核心工作事项
- 完成：A 项目数据分析与报告撰写，报告已同步给相关同事
- 推进中：B 功能方案设计与初稿编写
- 完成：协助处理线上紧急 Bug 并完成修复
二、工作成果/进展
- A 项目数据分析报告已产出并完成内部同步
- B 功能方案初稿已完成，待评审
- 线上 Bug 已定位并修复，系统运行恢复正常
三、问题/待协调事项
- A 项目数据源异常，需协调小李协助排查与处理
四、下周初步计划
- 根据评审意见优化 B 功能方案
- 启动 C 技术的可行性调研工作
```

**示例 2：含发送需求**

> 主管邮箱是 zhang.manager@company.com。这周完成了客户 D 的需求沟通，系统 E 版本上线，下周计划跟进客户反馈并准备下季度规划。

Agent 整理周报后调用 `send_email(to="zhang.manager@company.com", subject="工作周报 - [姓名] - [周期]", content=HTML邮件内容)` 发送。

**示例 3：内容极简**

> 这周在改 bug。下周继续。

Agent 不做内容扩充，仅做语句通顺化处理：
- 核心工作事项：推进系统缺陷修复
- 工作成果/进展：部分 Bug 已解决
- 问题/待协调事项：暂无
- 下周计划：根据本周推进情况，规划后续工作落地

**示例 4：抄送支持**

> 主管邮箱 li.leader@company.com，抄送 team@company.com。本周核心推进 F 项目一阶段开发，测试报告已出，遇到第三方接口性能问题，已联系对方技术支持。下周进入二阶段。

Agent 整理周报后调用 `send_email(to, subject, content, cc="team@company.com")` 发送。

**示例 5：模板问询**

> 想看一下标准的工作周报应该包含哪些部分？

Agent 直接回答周报标准结构（标题、四大模块、分级标题、列表排版），不调用工具。

## 核心原则

- **不杜撰**：严格基于用户输入整理，不额外添加信息
- **不过度扩充**：内容过简时仅做通顺化处理
- **量化优先**：工作成果优先用量化数据
- **不发陌生人**：必须先获得收件人邮箱再发送
- **HTML 排版**：邮件内容使用 HTML 格式，符合职场邮件规范
- **主题完整**：邮件主题需体现为工作周报 + 姓名 + 周期
- **支持 CC**：支持抄送字段
