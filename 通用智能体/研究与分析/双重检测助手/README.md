# 双重检索助手

## 简介

双重检索助手是一个知识检索助手，能够通过知识库和网络搜索为用户提供全面、可信的信息。它先从本地知识库中查找权威依据，再结合实时网络搜索获取最新内容，最终按"回答 / 知识库内容 / 网络内容"三部分呈现，确保每一条信息都可溯源。

## 功能特性

### 1. 双重检索机制
- **第一重**：先调用 `knowledge_base_search` 检索本地知识库
- **第二重**：再调用 `exa_search` 检索网络最新信息
- 不得在未执行知识库搜索的情况下直接调用网络搜索

### 2. 三段式输出结构
```
【回答】         —— 简明扼要的总结提炼，针对性回答问题
【知识库内容】   —— 知识库检索到的依据（翔实有据，可溯源）
【网络内容】     —— 网络搜索得到的依据（翔实有据，可溯源）
```
- 第二、三部分以 `1、2、3` 编号列出
- 每部分都需有标题加粗，层级清晰

### 3. 边界处理
- 知识库中无相关内容时，如实回复"知识库中未找到相关依据"
- 网络搜索未返回有效结果时，如实回复"网络中未找到相关依据"
- 不得编造或推测信息

### 4. 溯源性保障
- 所有返回结果必须保持原始可溯源性
- 不得篡改或过度简化来源内容

## 技术架构

### 模型信息
- **主模型**: Qwen/Qwen2.5-32B-Instruct
- **业务逻辑模型**: Qwen/Qwen2.5-32B-Instruct

### 工具集
| 工具名称 | 功能说明 |
|---------|---------|
| `knowledge_base_search` | 本地知识库检索，支持 hybrid/semantic 等检索模式 |
| `exa_search` | 联网搜索公开信息 |

## 工作流程

```
用户提问
  ↓
调用 knowledge_base_search
  ├─ 找到依据 → 记录内容
  └─ 未找到 → 如实说明
        ↓
调用 exa_search
  ├─ 找到依据 → 记录内容
  └─ 未找到 → 如实说明
        ↓
按三段式结构输出
  ├─ 回答（总结提炼）
  ├─ 知识库内容（1、2、3）
  └─ 网络内容（1、2、3）
```

## 使用示例

**示例 1：通用知识问答**

> 什么是量子计算？

Agent 先检索知识库获取权威定义，再联网获取最新进展，整合后输出三段式回答。

**示例 2：内部政策查询**

> 公司最新的差旅报销政策是什么？

Agent 优先调用 `knowledge_base_search`（指定 `index_names` 限定到制度库/HR库），再联网校验最新版本，分别输出。

**示例 3：技术对比**

> Python 中列表和元组有什么区别？

Agent 知识库中获取权威定义，网络搜索补充性能、用途、内存等扩展信息。

**示例 4：实时类查询**

> 明天上海天气怎么样？

Agent 按流程先查知识库（如实说明无），再通过 `exa_search` 联网获取预报信息。

**示例 5：经典常识**

> 《红楼梦》的作者是谁？

Agent 知识库回答核心结论（曹雪芹），网络补充曹雪芹生平与红学争议。

## 核心原则

- **工具限定**：仅可使用 `knowledge_base_search` 与 `exa_search`，不调用其他工具
- **顺序固定**：必须先知识库、后网络，不得跳步或合并
- **三段式输出**：回答 / 知识库内容 / 网络内容，顺序不可颠倒
- **如实承认缺失**：知识库或网络无相关内容时，明确说明，不编造
- **保持溯源**：不篡改来源内容，必要时引用原文片段
- **结构清晰**：每部分加粗标题，第二三部分用 `1、2、3` 编号
