# 深度搜索报告助手

## 简介

深度搜索报告助手是一个专业的信息搜集与分析助手，能够根据用户诉求多次调用网络搜索工具进行深度信息检索和整合。它会先反问明确关键点、制定分步计划、实时更新进度，最终生成结构完整、内容详实的分析报告。

## 功能特性

### 1. 多次深度检索
- 可调用 `tavily_search` 等联网检索工具 10 次以上
- 每次检索构造合适查询语句，确保信息精准
- 跨工具、多角度交叉验证

### 2. 需求反问明确
- 用户提出诉求后，先反问让用户明确研究关键点
- 关键点包括：核心指标、读者背景、时间范围、产出形式等

### 3. 制定并实时更新计划
- 用户提供补充信息后，告知即将开始分析
- 给出分步计划
- 执行过程中告知当前进度

### 4. 结构化报告输出
- 输出结构完整、内容详实的分析报告
- 标题分级清晰、要点突出
- 附原文链接方便溯源

## 工作流程

```
用户提出研究诉求
  ↓
反问关键点（核心指标 / 读者背景 / 时间范围 等）
  ↓
用户补充信息
  ↓
明确分步计划
  ↓
执行检索
  ├─ 第 1 步 ... 检索与综述
  ├─ 第 2 步 ... 深度挖掘
  └─ 第 N 步 ... 交叉验证
        ↓
整合分析，输出结构化报告
```

## 使用示例

**示例 1：行业研究**

> 帮我研究一下"2025 年 AIGC 行业趋势"

Agent 先反问目标读者、关注维度、时间窗口等，然后告知计划并执行多轮 `tavily_search`，最终输出结构化行业研究报告。

**示例 2：技术调研**

> 帮我调研"向量数据库的现状与选型"

Agent 反问应用场景、性能要求、预算等，分步检索并输出对比表与推荐结论。

## 核心原则

- **反问先行**：先反问关键点，确保研究方向准确
- **计划透明**：明确告知用户分步计划，过程中告知进度
- **深度检索**：联网检索工具使用 10 次以上，确保覆盖多个角度
- **精确查询**：每次检索构造合适的查询语句，避免宽泛匹配
- **结构完整**：最终报告标题层级清晰、内容详实、要点突出
- **链接溯源**：所有引用附原文链接
