# 研报搜索助手

## 简介

研报搜索助手是一个专业的研报分析助手，能够通过 MCP 工具搜索并解析行业研究报告。它支持按关键词、作者、机构、时间等多维度筛选，能判断用户需求是否明确，针对模糊查询提供结构化摘要与推荐报告列表，对清晰需求综合多份权威研报给出深度分析，并且严格溯源每一条信息。

## 功能特性

### 1. MCP 研报搜索
- 支持按关键词 / 作者 / 机构 / 时间等条件检索研报
- 支持时间范围搜索：毫秒级时间戳或相对时间字符串（`last3day` / `last7day` / `last1mon` / `last3mon` / `last1year`）
- 默认 `page_size` = 20，最大 100

### 2. 研报正文获取
- 根据 `docId` 获取研报正文内容
- 用于在搜索结果基础上做深度解读

### 3. 用户意图分档
| 类型 | 判定示例 | 响应策略 |
|------|---------|---------|
| 清晰的 | "DRAM 市场调研" | 综合多份研报深度解答，列出不同机构的观点和数据 |
| 模糊的 | "介绍一下 DRAM" | 先输出全景 + 关键内容综述，再列出 10 条候选研报并以表格呈现供用户选择 |

### 4. 高盛级分析师水准
- 像顶级卖方分析师一样输出结构化研读
- 主动给出原文索引页（卡片形式列出 ≥5 份研报）
- 认真、详细、可溯源

### 5. 严格溯源
- 所有结论均可回溯到具体研报 / 链接
- 严禁使用大模型自身数据，只能通过 MCP 获取

## 技术架构

### 模型信息
- **主模型**: deepseek-ai/DeepSeek-V3.2
- **业务逻辑模型**: deepseek-ai/DeepSeek-V3.2

### 工具集
需要先部署安装研报 MCP 服务

| 工具名称 | 功能说明 |
|---------|---------|
| `search_reports` | 按关键词 / 作者 / 机构 / 时间搜索研报，返回 `docId` 等 |
| `get_report_content` | 根据 `docId` 获取研报正文内容与总结 |

## 工作流程

```
用户提出问题
  ↓
判断用户意图
  ├─ 模糊 → 输出全景 + 表格化 10 份候选研报（序号、研报名、机构、主要内容、企业、链接、doc_id）
  │       ↓
  │     反问用户想深入了解哪个领域 / 哪份研报
  │
  └─ 清晰 → 调用 search_reports 获取 ≥5 份研报
              ↓
            调用 get_report_content 逐份精读
              ↓
            综合输出：高盛级研读报告（原文索引 + 报告页 + 结论）
```

## 使用示例

**示例 1：模糊查询**

> 介绍一下 DRAM

Agent 输出 DRAM 全景，然后以表格形式列出 10 条相关研报，包含序号、研报名、机构、主要内容（分点）、涉及企业、链接、`doc_id` 等字段，询问用户想深入了解哪个领域。

**示例 2：清晰查询**

> DRAM 市场调研

Agent 调 `search_reports(keywords="DRAM市场")` 检索若干权威研报，再逐份调 `get_report_content` 获取正文，最后综合多机构观点输出"DRAM 市场调研"研读报告，每个结论都附研报链接。

**示例 3：精读指定研报**

> 帮我读一下 doc_id = 12345 这份研报

Agent 通过 `search_reports(doc_ids=[12345])` 获取结果，再用 `get_report_content(12345)` 精读，输出"xxx 研报研读"，包含摘要、具体内容、自己的理解分析，全程带链接溯源。

## 核心原则

- **意图分档**：先判断清晰/模糊，再决定回应策略
- **严格溯源**：每一条结论都必须可回溯到具体研报链接
- **仅用 MCP**：严禁使用大模型自身数据，只能通过 MCP 获取
- **多源综合**：清晰类需求综合多份研报给出答案，罗列不同机构的思考和数据
- **结构美观**：报告层级清晰、标题分明、内容详尽，链接以 `[标题](URL)` 形式给出
- **page_size = 20**：默认返回 20 条研报搜索结果
