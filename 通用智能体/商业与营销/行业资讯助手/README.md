# 行业资讯助手

## 简介

行业资讯助手是一个行业洞察工具，具备获取行业热门资讯、搜索特定新闻以及生成每日简报的能力。它能够灵活管理行业信息源，每天为用户精准推送最新的行业动态与深度资讯，并输出可直接下载的 Markdown 格式报告。

## 功能特性

### 1. 行业资讯获取
- 通过 `fetch_top_news` 获取指定行业的最新热门资讯
- 支持全局、战略、产品、技术四种视图筛选
- 通过 `list_industries` 查看支持的行业列表
- 通过 `search_news` 按关键词或厂商检索历史资讯

### 2. 智能摘要生成
- 对每条新闻通过 `exa_search` 联网检索补充背景信息
- 为每条资讯生成 100-200 字的中文摘要
- 摘要中以 Markdown 链接形式标注信息来源，方便溯源

### 3. 资讯分类管理
| 功能 | 说明 |
|------|------|
| 行业细分 | 按子领域分类组织资讯（如生物制药、医疗 AI 等） |
| 多视图 | 全局/战略/产品/技术四种视角 |
| 源管理 | 支持查看和新增 RSS 源配置 |

### 4. 报告输出
- 资讯按行业细分分类后组织成结构化报告
- 每条资讯保留原始链接供用户跳转查看
- 调用 `create_file` 输出可下载的 Markdown 文件

## 技术架构

### 模型信息
- **主模型**: glm-5.1
- **业务逻辑模型**: glm-5.1

### 工具集
需要先部署安装mcp服务（mcp_industry_news）

| 工具名称 | 功能说明 |
|---------|---------|
| `fetch_top_news` | 获取指定行业 Top-N 缓存新闻 |
| `search_news` | 按关键词/厂商搜索缓存新闻 |
| `exa_search` | 联网搜索补充新闻背景信息 |
| `generate_daily_brief` | 生成格式化每日简报（辅助参考） |
| `list_industries` | 查看所有支持的行业 |
| `list_sources` | 查看行业配置的 RSS 源 |
| `add_source` | 动态新增 RSS 源 |
| `create_file` | 创建可下载的 Markdown 报告文件 |

## 工作流程

```
用户指定行业
  ↓
fetch_top_news 获取最新资讯
  ↓
对每条新闻 exa_search 联网检索背景信息
  ↓
撰写 100-200 字摘要（含来源链接）
  ↓
按行业子领域分类整理
  ↓
create_file 输出可下载的 Markdown 报告
```

## 使用示例

**示例 1：推送行业最新资讯**

用户输入：
```
给我推送今天存储行业的最新资讯
```

Agent 调用 `fetch_top_news` 获取存储行业热门新闻，对每条新闻通过 `exa_search` 补充背景信息并撰写摘要，按"AI 存储"、"企业存储"等子领域分类，最终生成 Markdown 报告供下载。

**示例 2：关键词检索**

用户输入：
```
看看最近两周存储行业关于"戴尔"的相关新闻
```

Agent 调用 `search_news` 按关键词检索近 14 天与戴尔相关的存储资讯，补充背景摘要后生成专题报告。

**示例 3：查看支持的行业**

用户输入：
```
查询一下系统目前都支持哪些行业的资讯推送
```

Agent 调用 `list_industries` 查看支持的行业列表（如存储、生物制药、医疗 AI 等）。

## 核心原则

- **缓存优先**：默认读取本地缓存数据快速响应；仅在用户明确要求且能接受等待时才刷新
- **搜索补充**：每条新闻必须通过 `exa_search` 补充背景信息，撰写带来源标注的摘要
- **分类呈现**：资讯需按行业子领域分类后组织输出
- **保留链接**：必须保留每条新闻的原始链接，方便用户溯源
- **文件输出**：完成报告后必须调用 `create_file` 输出可下载的 Markdown 文件
- **源管理谨慎**：仅在有明确需求时使用 `add_source`，不擅自修改行业源配置
