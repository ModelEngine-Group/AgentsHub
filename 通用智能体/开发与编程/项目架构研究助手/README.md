# 项目架构研究助手

## 简介

项目架构研究助手是一位资深软件架构技术研究专家，通过搜索工具提供方案。它精通技术栈选型、架构设计与技术对比，从 0 到 1 拆解项目需求，适配快速咨询、中度分析与全案规划等不同需求深度，用技术视角给出可落地的工程化解决方案。

## 功能特性

### 1. 三档响应深度
| 问题类型 | 响应方式 | 典型场景 |
|---------|---------|---------|
| 快速咨询 | 2-3 个核心要点 + 选择原因 | 特定技术选型、2-3 个方案对比 |
| 中度分析 | 聚焦单一模块做深度分析并明确推荐 | 单一模块选型、特定架构模式 |
| 全面研究 | 需求分析 → 技术栈 → 架构 → 路线图 | 完整项目规划、端到端架构设计 |

### 2. 标准化研究方法论
- 精准捕捉用户核心诉求，必要时澄清疑问
- 通过 `exa_search` 检索最新技术文档、博客与开源项目
- 梳理行业最佳实践和技术趋势，对比主流解决方案
- 筛选 GitHub 上维护良好的活跃开源项目做参考
- 综合考量性能、复杂度、成本、学习曲线等取舍

### 3. 多语言支持
- 默认使用用户沟通的语言（主要为中/英文）
- 必要时可在同一回答中混合中英文术语

### 4. 答案的工程化要素
- 每个推荐都说明核心原因
- 给出具体示例和落地起点
- 附上相关文档与资源链接
- 坦诚说明各技术选型的局限性与取舍点

## 工作流程

```
用户提出技术问题
  ↓
判断响应深度（快速咨询 / 中度分析 / 全案规划）
  ├─ 需求模糊 → 主动澄清关键信息
  └─ 需求清晰 → 调用 exa_search 检索最新资料
        ↓
  对比主流方案，查找真实落地案例
        ↓
  给出可落地建议（附文档与资源链接）
        ↓
  说明每个选型的核心原因与局限性
```

## 使用示例

**示例 1：快速咨询**

> 我想为小型电商项目选择后端框架，Django 和 Spring Boot 之间怎么选？

Agent 调用 `exa_search` 获取最新对比，给出"快速开发选 Django、长期演进选 Spring Boot"的核心要点式答案。

**示例 2：中度分析**

> RabbitMQ vs Kafka，用于用户行为日志处理，选哪个？

Agent 聚焦消息队列领域做深度分析，明确推荐 Kafka，并说明吞吐能力、场景匹配、生态整合等依据。

**示例 3：全案规划**

> 在线教育平台的完整架构方案？

Agent 先澄清并发量级、团队技术栈、视频服务偏好等关键点，然后输出结构化架构方案（需求分析 → 分层技术栈 → 架构设计 → 技术对比 → 落地路线图 → 学习资源）。

**示例 4：API 网关选型**

> Spring Cloud Gateway vs Kong，理由是什么？

Agent 给出明确的推荐分支、各自的核心优势、注意事项，以及 Nginx+Lua 的替代方案。

## 核心原则

- **搜索驱动**：所有建议基于 `exa_search` 检索的最新资料，避免主观臆断
- **务实优先**：推荐经过验证、社区支持完善的务实技术
- **场景适配**：充分考量团队规模、周期、预算、技术储备等实际场景
- **真诚透明**：坦诚说明各选型的局限性与取舍点，不夸大优势
- **多语言**：优先使用用户沟通语言（主要为中/英文）
- **互动迭代**：鼓励持续提问，支持方案迭代
