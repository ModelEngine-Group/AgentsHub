# 商品折扣助手

## 简介

商品折扣助手是一个智能购物工具，能够搜索商品折扣信息并抓取网页详情，为每条折扣整理出对应的购买方案。遇到反爬虫时自动降级返回基础折扣与链接，并智能去重同款商品、保留同类或配件的优惠信息。

## 功能特性

### 1. 商品折扣搜索
- 通过 `searchZheKou` 工具搜索任意商品的折扣信息
- 覆盖多平台（官网、京东、天猫、拼多多等）
- 实时获取最新的促销活动

### 2. 购买方案整理
- 对每条折扣信息抓取详情页面
- 提取具体的购买方式、优惠条件、使用限制
- 每条折扣严格对应一条购买方案

| 折扣示例 | 购买方案 |
|---------|---------|
| 苹果官网教育优惠直降 500 元 | 验证学生身份，官网下单，支持 24 期免息 |
| 京东满 5000 减 200 | 领取满减券，结算时自动抵扣 |
| 拼多多百亿补贴降价 600 元 | 直接拼单购买，发货时间可能较长 |

### 3. 智能去重与筛选
- **同款商品**：最多保留 3 条折扣信息，避免过多重复
- **同类产品**：不同型号/规格的同类商品予以保留
- **配件推荐**：相关配件（如手柄、游戏、保护壳）的折扣信息一并保留

### 4. 反爬虫降级处理
- 遇到反爬虫机制导致抓取失败时，不强行重试
- 自动降级为基础模式：返回折扣摘要和原始链接
- 确保用户至少能获取到折扣线索

## 技术架构

### 模型信息
- **主模型**: deepseek-v4-pro
- **业务逻辑模型**: deepseek-v4-pro

### 工具集
| 工具名称 | 功能说明 |
|---------|---------|
| `searchZheKou` | 搜索商品折扣信息，返回折扣标题、摘要和 URL |
| `fetch_html` | 抓取折扣详情页面的 HTML 内容，提取购买方案 |

### 外部集成
| 服务 | 用途                |
|------|-------------------|
| **Shopping MCP** | 折扣搜索引擎（来源慢慢买比价平台） |
| **Fetch URL MCP** | 网页内容抓取引擎          |

## 工作流程

```
用户输入商品名称
  ↓
searchZheKou 搜索折扣信息
  ↓
智能去重：
  ├─ 同款商品 → 最多保留3条
  ├─ 同类/配件 → 全部保留
  ↓
对每条折扣 fetch_html 抓取详情
  ↓
成功？→ 提取购买方案（优惠条件/使用方式）
  ↓
失败（反爬虫）？→ 降级返回基础折扣+链接
  ↓
整理输出：折扣信息 + 购买方案一一对应
```

## 使用示例

**示例 1：正常抓取**

用户输入：
```
帮我找一下iPhone 15的折扣信息
```

Agent 搜索到 3 条折扣（苹果官网教育优惠、京东满减、拼多多百亿补贴），成功抓取各页面详情，整理出 3 条完整的购买方案（含验证方式、优惠券领取、分期信息）。

**示例 2：反爬虫降级**

用户输入：
```
有没有戴森吹风机的优惠？
```

Agent 搜索到 2 条折扣，但抓取天猫详情时触发反爬虫（403 Forbidden），降级返回基础折扣信息和原始链接供用户自行查看。

**示例 3：同款去重+配件保留**

用户输入：
```
我想买索尼PS5，看看有什么折扣
```

Agent 搜索到 6 条结果（4 条主机 + 1 条手柄 + 1 条游戏），主机同款只保留 3 条，手柄和游戏折扣保留，最终输出 5 条折扣及对应购买方案。

## 核心原则

- **先搜后抓**：严格按顺序先 `searchZheKou` 再 `fetch_html`，`fetch_html` 的 URL 必须来源于搜索结果
- **同款去重**：同款商品最多保留 3 条折扣
- **配件保留**：同类产品或配件折扣不受 3 条限制
- **反爬降级**：抓取失败时不重试，返回基础折扣和链接
- **一一对应**：每条折扣必须对应一条购买方案
