# 【医疗】用药科学规划助手

## 简介

用药科学规划助手是一个专业的智能用药管理 AI 代理，具备药品查询、用药方案审核和个性化用药提醒三大核心能力。能够用通俗语言解释药品信息，检查用药冲突与禁忌症，制定个性化用药时间表，并通过微信实时推送服药提醒。

## 功能特性

### 1. 药品智能查询
- 输入药品名，详细说明功能主治、用法用量、禁忌症、不良反应、药物相互作用
- 支持 `tavily_search` 联网搜索最新药品信息
- 用通俗语言解释，不确定的信息主动说明

### 2. 用药方案审核
- 逐个核对每种药是否对症
- 检查药品之间的相互作用和冲突
- 检查禁忌症风险（过敏史、肝肾等）
- 给出综合评估和潜在风险提示

### 3. 个性化用药提醒
- 根据用户作息制定每日用药时间表
- 标注饭前/饭后、间隔时间等注意事项
- 后台每 30 秒自动扫描，到点通过 Server酱推送到微信

### 4. 用户流程
1. 用户输入症状和正在服用的药品
2. Agent 调用 `list_medications` 获取当前用药清单
3. 逐药核对对症性，联网检查药物相互作用
4. 制定个性化用药时间表
5. MCP 后台自动推送微信提醒

## 技术架构

### 模型信息
- **主模型**: DeepSeek-Chat

### 工具集
| 工具名称 | 功能说明 | 来源 |
|---------|---------|------|
| `tavily_search` | 联网搜索药品信息、药物相互作用 | Nexent 内置 |
| `list_medications` | 列出当前所有用药计划 | MCP |
| `add_medication` | 添加新药品到用药计划 | MCP |
| `stop_medication` | 暂停某个药品 | MCP |
| `resume_medication` | 恢复已暂停的药品 | MCP |
| `update_medication` | 修改药品信息 | MCP |
| `send_test_notification` | 测试微信推送是否正常 | MCP |
| `check_now` | 立即检查是否有到点的用药并推送提醒 | MCP |

### MCP 服务
| 参数 | 值 |
|------|-----|
| 服务名称 | medication-reminder |
| 端口 | 5060 |
| 协议 | Streamable HTTP |
| 后台扫描间隔 | 30 秒 |

## 使用示例

**示例 1：药品查询**

用户输入：
```
帮我查一下阿莫西林的详细信息
```

Agent 响应：
- **功能主治**：敏感细菌引起的感染（呼吸道、泌尿系统等）
- **用法用量**：成人一次 0.5g，每 6-8 小时一次，饭后服用
- **禁忌症**：青霉素过敏者禁用
- **不良反应**：腹泻、恶心、皮疹
- **药物相互作用**：与丙磺舒、甲氨蝶呤有相互作用

**示例 2：用药审核**

用户输入：
```
我感冒了，在吃感康和头孢克肟，帮我审核一下用药是否合理
```

Agent 首先获取当前用药清单，然后逐药核对：感康 → 对症（缓解感冒症状）；头孢克肟 → 普通感冒多为病毒引起，不建议常规使用；两者无严重冲突。综合建议：除非有细菌感染证据，建议停用头孢克肟。

**示例 3：用药提醒**

用户输入：
```
添加阿莫西林 0.5g，每天 8:00/12:00/20:00，饭后服用，一个疗程 7 天
```

Agent 先检查是否与现有用药冲突，确认后调用 `add_medication` 添加。MCP 后台自动在 8:00、12:00、20:00 通过微信推送提醒。

## 部署说明

### 前置条件
- Nexent 平台已部署运行
- Python 3.10+
- [Tavily Search](https://tavily.com) API Key（免费额度每月 1000 次）
- [Server酱](https://sct.ftqq.com) SendKey（微信扫码即得，不配也能用 CRUD，只是没有微信推送）

### 第一步：导入 Agent 配置

1. Nexent → 智能体管理 → 导入
2. 选择本目录下的 `agent.json`
3. 导入后可在智能体列表中看到「用药科学规划助手」

### 第二步：配置 API Key

导入后在 Nexent 智能体编辑页 → 工具配置 → `tavily_search` → 修改参数：

| 参数 | 当前值 | 改为 |
|------|--------|------|
| `tavily_api_key` | `YOUR_TAVILY_API_KEY_HERE` | 你的 Tavily API Key |

### 第三步：部署 MCP 服务

1. 克隆源码：
```bash
git clone https://gitcode.com/gcw_ngCW84xk/medication-server.git
cd medication-server
```

2. 配置微信推送（可选）：
```bash
cp reminder_config.example.json reminder_config.json
# 编辑 reminder_config.json，填入你的 Server酱 SendKey
```

3. 安装依赖并启动：
```bash
pip install mcp
python medication_mcp_server.py
```

启动后输出：
```
Medication MCP Server v1.0
MCP URL:  http://0.0.0.0:5060/mcp
```

### 第四步：注册 MCP 服务

1. Nexent → MCP 管理 → 添加远程 MCP 服务
2. 名称：`medication-reminder`
3. URL：`http://localhost:5060/mcp`（Docker 部署时用 `http://host.docker.internal:5060/mcp`）
4. 保存

### 第五步：开始使用

在 Nexent 聊天界面选择「用药科学规划助手」，试试这些话：

- "帮我查一下阿莫西林的详细信息"
- "我最近感冒了，正在吃感康和头孢克肟，帮我审核一下用药是否合理"
- "帮我添加阿莫西林 0.5g，每天三次 8:00/12:00/20:00，饭后服用，一个疗程7天"
- "看看我现在的用药清单"
- "帮我暂停布洛芬"

## 核心原则

- **安全第一**：每次回答健康/用药问题强制加免责声明
- **先查后加**：添加新药前必须先查看现有用药，检查重复和冲突
- **冲突必报**：发现药物相互作用、过敏风险、禁忌症立即警告
- **征求同意**：操作前征得用户同意后再执行

> ⚠️ **免责声明**：本智能体仅供健康参考，不构成专业医疗诊断，不能替代医生、药师等专业人员。用药请遵医嘱。
