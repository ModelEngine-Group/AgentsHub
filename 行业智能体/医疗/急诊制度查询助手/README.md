# 【医疗】急诊制度查询助手

## 简介

急诊制度查询助手是一个智能 AI 代理，基于急诊制度及法规，为用户提供安全、合规的急诊医疗信息查询服务。通过调用知识库进行精确检索，严格引用原文回答问题，确保信息的准确性和可靠性。

## 功能特性

### 1. 急诊规范查询
- 支持查询急诊患者入院初步检查项目
- 支持查询危急值报告流程与要求
- 支持查询急诊处方安全要求
- 支持查询急诊医保使用规范
- 支持查询患者双向核对流程

### 2. 合规与安全底线提示
在涉及合规或安全底线问题时，自动提示以下关键要求：
- **执业资格**：确认相关人员的执业资质
- **医保规范**：遵循医保使用规定
- **用药安全**：核对药物过敏史、剂量、频次
- **患者双向核对**：确认患者身份信息
- **危急值报告**：及时报告并采取紧急措施
- **不良事件上报**：按规定流程上报

### 3. 严格溯源引用
- 所有回答严格引用知识库原文
- 不添加、修改或总结原文内容
- 无法找到依据时明确说明无依据

### 4. 用户流程
1. 用户提出急诊相关问题
2. 自动判断问题类型（一般领域 / 合规安全底线）
3. 调用知识库检索相关内容
4. 严格引用原文生成回答
5. 若涉及合规底线，附加关键安全提示

## 使用示例

**示例 1：查询急诊初步检查**

用户输入：
```
急诊患者在入院时需要进行哪些初步检查？
```

Agent 响应：
> 根据《急诊制度及法规》规定，急诊患者入院时需要进行的初步检查包括：生命体征监测、病史询问、体格检查、必要的实验室检查（如血常规、尿常规）和影像学检查（如X光、CT）。

**示例 2：查询危急值报告**

用户输入：
```
在急诊中，哪些情况下需要报告危急值？
```

Agent 响应：
> 根据《急诊制度及法规》规定，在急诊中，当患者出现生命体征异常、重要实验室检查结果超出正常范围、或存在潜在生命危险时，必须立即报告危急值，并采取相应紧急措施。

**示例 3：查询处方安全要求**

用户输入：
```
急诊医生在开具处方时有哪些安全要求？
```

Agent 响应：
> 根据《急诊制度及法规》规定，急诊医生在开具处方时必须严格遵守用药安全要求，包括但不限于：确认患者身份、核对医嘱、检查药物过敏史、遵循用药剂量和频次、以及记录患者用药反应等。
