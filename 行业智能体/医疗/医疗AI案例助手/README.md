# 【医疗】医疗 AI 案例助手

## 简介

医疗 AI 案例助手是一个医疗行业案例生成代理，能够围绕指定公司梳理其 AI 应用案例，从应用场景、技术能力、业务价值等方面进行分析，帮助用户全面了解该公司在医疗领域的 AI 实践。

## 功能特性

### 1. 公司 AI 案例梳理
- 支持按公司维度梳理 AI 应用全景
- 覆盖医疗影像、药物研发、手术机器人、医院智能运营等场景
- 整合公开资料中的案例信息，形成结构化报告

### 2. 多维度分析框架
| 维度 | 说明 |
|------|------|
| **应用场景** | 具体医疗环节中的 AI 落地场景 |
| **技术能力** | 所用 AI 技术、算法、产品方案 |
| **业务价值** | 效率提升、成本降低、质量改善等成效 |
| **客户痛点** | 待解决的业务瓶颈与需求 |
| **落地效果** | 公开报道中展示的实际成果 |

### 3. 公开信息检索与验证
- 通过 `exa_search` 检索公司官网、新闻报道、行业报告等公开资料
- 优先采用权威来源（官网、权威媒体、监管机构、学术机构）
- 多来源交叉验证，确保信息可靠性
- 公开资料不足时如实说明，不编造细节

### 4. 合规输出
- 不使用搜索工具获取非公开信息、商业机密、患者隐私或受保护医疗数据
- 涉及 AI 医疗产品疗效、合规认证、监管批准等信息时，严格基于可核验公开来源
- 对未经充分验证的信息使用审慎表述（如"据公开资料显示""公开报道提到"等）

## 使用示例

**示例 1：医疗影像 AI 案例**

用户输入：
```
帮我整理西门子医疗在AI医学影像方面的应用案例，重点说明应用场景、解决方案和业务价值
```

Agent 检索公开资料后梳理出西门子医疗 AI-Rad Companion 方案，覆盖胸部 CT、脑部影像、肿瘤评估等场景，通过 AI 算法自动识别关键结构、辅助发现异常病灶并生成定量分析结果。

**示例 2：医院智能运营 AI 案例**

用户输入：
```
请帮我生成GE HealthCare在医院智能运营中的AI应用案例，要求包含客户痛点、AI能力和落地效果
```

Agent 检索发现 GE HealthCare 在 Command Center、智能影像工作流、设备运营优化等方面的 AI 应用，涵盖患者流转效率低、床位资源紧张、检查排程复杂等痛点。

**示例 3：手术机器人案例**

用户输入：
```
帮我分析强生在手术机器人和AI辅助外科领域的应用案例，适合写进医疗AI行业报告
```

Agent 围绕强生数字化手术平台和手术机器人展开，覆盖术前规划、术中图像引导、手术器械辅助控制和术后数据复盘等场景。

**示例 4：AI 药物研发案例**

用户输入：
```
请生成辉瑞使用AI进行药物研发的案例，重点关注AI如何提升研发效率和降低失败风险
```

Agent 检索到辉瑞在靶点发现、分子筛选、临床试验设计和真实世界数据分析中使用 AI，展现 AI 在提升研发效率、减少无效实验、降低临床失败风险方面的价值。

## 工作流程

```
用户指定公司 → exa_search 检索公开资料
    → 多来源交叉验证
    → 结构化分析（场景/技术/价值）
    → 生成案例报告
```

## 核心原则

- **公开可验**：所有信息源自可核验的公开资料，不依赖模型记忆生成案例
- **来源优先**：官网 > 权威媒体 > 监管/学术机构 > 可信行业报告
- **交叉验证**：多方来源相互印证，避免单一信源偏差
- **不编造**：公开资料不足时如实说明，不虚构细节
- **审慎表述**：未充分验证的信息使用"据公开资料显示"等表述
- **合规安全**：不检索非公开信息、商业机密、患者隐私或受保护医疗数据
