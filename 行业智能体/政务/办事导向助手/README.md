# 办事向导助手

## 简介

办事向导助手（"办事小向导"）是一个专注于政务办事流程咨询的智能助手，能够解答各类政务办事流程问题。具备文档解析、知识库检索和联网搜索能力，可自动生成清晰的流程图，提供分步指引、温馨提示和参考来源。

## 功能特性

### 1. 多源信息检索（优先级机制）
按以下优先级自动选择信息来源：

| 优先级 | 来源 | 触发条件 |
|--------|------|---------|
| 1 | **文档解析** | 用户上传了办事指南文档 |
| 2 | **知识库搜索** | 未上传文档，检索"政务办事流程"知识库 |
| 3 | **联网搜索** | 前两者均未命中，且涉及具体办事流程 |

命中一项即停止后续检索，确保高效。

### 2. 流程图自动生成
- 涉及步骤、流程、办理、申请等关键词时，自动生成 Mermaid 流程图
- 支持 `png_url` / `svg_url` / `base64` 等多种输出格式
- 纯咨询类问题无需流程图

### 3. 结构化回答模板

```
📌 信息来源：{文档解析/知识库搜索/联网搜索}

📝 回答正文：
（分点列出办理步骤）

📊 办理流程：
[流程图图片]

💡 温馨提示：
（一句话建议）

🔗 参考来源：
（链接或文档说明）
```

## 技术架构

### 模型信息
- **主模型**: deepseek-v4-pro
- **业务逻辑模型**: deepseek-v4-pro

### 工具集
| 工具名称 | 功能说明 |
|---------|---------|
| `analyze_text_file` | 解析用户上传的办事指南文档（PDF 等），提取办理步骤 |
| `knowledge_base_search` | 检索"政务办事流程"知识库，获取已入库的办事指南 |
| `exa_search` | 联网搜索最新办事指南和官方政策 |
| `generate_mermaid_diagram` | 使用 Mermaid 语法生成办理流程图（推荐） |
| `generate_flow_diagram` | 生成流程图表（备选方案） |

### 外部集成
| 服务 | 用途 |
|------|------|
| **Mermaid MCP** | 流程图生成引擎 |
| **Vis MCP** | 备选图表生成引擎 |

## 使用示例

**示例 1：知识库检索 + 流程图**

用户输入：
```
我想了解一下在上海怎么办理居住证？
```

Agent 检索知识库，返回居住证办理 6 步流程，自动生成流程图，并提示"建议提前致电社区事务受理中心确认所需材料清单"。

**示例 2：文档解析 + 流程图**

用户上传《护照办理须知.pdf》并询问怎么办护照。

Agent 优先解析文档，提取 6 步办理流程（网上预约 → 现场拍照 → 缴费 160 元 → 10 个工作日后领取），生成流程图，标注信息来源为文档解析。

**示例 3：联网搜索 + 流程图**

用户询问结婚登记办理流程，知识库未命中，Agent 自动联网搜索民政部官网信息，完成回答并生成流程图，同时提示"全国已推行'跨省通办'"。

**示例 4：纯咨询类（无需流程图）**

用户询问"社保卡有什么用？"，Agent 检索知识库后列出 6 项功能（医保结算、养老金领取、身份凭证等），无需生成流程图。

## 工作流程

```
用户提问
  ├─ 有上传文档？→ analyze_text_file → 生成回答
  ├─ 无文档？→ knowledge_base_search → 命中？→ 生成回答
  └─ 均未命中且涉及流程？→ exa_search → 生成回答
      ↓
  涉及步骤流程？→ generate_mermaid_diagram → 附加流程图
      ↓
  按模板输出（信息来源 + 正文 + 流程图 + 温馨提示 + 参考来源）
```

## 核心原则

- **文档优先**：用户上传文档时优先解析，不跳过文档直接检索
- **单源回答**：同一回答只使用一个信息来源，命中即止
- **流程必图**：涉及办理步骤时强制生成流程图
- **真实引用**：参考来源如实标注出处，不编造
- **如实告知**：无法获取信息时建议用户咨询官方渠道
