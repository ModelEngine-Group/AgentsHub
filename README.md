## 智能体分类

### 🧩 通用智能体

| 分类 | 说明 |
|------|------|
| **研究与分析** | 面向信息检索、数据整理、报告撰写等场景，帮助用户快速获取并结构化处理信息 |
| **内容创作** | 各类文案、脚本、图文内容的生成与优化，提升内容生产效率 |
| **开发与编程** | 辅助代码编写、代码审查、技术文档生成、Bug 定位与修复建议 |
| **商业与营销** | 市场分析、客户洞察、营销策划、销售辅助等商业场景 |
| **自动化与工作流** | 日常重复性任务的自动化编排与执行，跨系统流程串联 |
| **教育与培训** | 学习辅助、知识点讲解、考题生成、培训材料制作 |
| **沟通与支持** | 客服应答、邮件撰写、多语言翻译、对内对外沟通辅助 |
| **数据处理** | 数据清洗、格式转换、抽取聚合、可视化前的预处理等工作 |
| **创意与设计** | 创意发散、方案构思、命名建议、风格探索等需要发散思维的场景 |
| **其他** | 不属于上述分类的通用智能体 |

## 快速使用

### 前置条件

- 可访问的 Nexent 平台实例
- 具备智能体开发功能的使用权限

### 导入智能体

1. 登录 Nexent 平台
2. 进入 **智能体开发** → **导入**
3. 上传智能体目录中的 `agent.json` 文件
4. 平台自动解析配置并创建智能体
5. 如有依赖的自定义 Skill 或 MCP（见目录中 `skills/` 或 `mcps/` 文件夹），按说明预先配置

## 开发指南

### 智能体结构

一个标准的 Nexent 智能体包含以下文件：

```
my-agent/
├── agent.json             # 智能体描述文件（Nexent 平台导出生成）
├── README.md              # 使用说明与配置指引（推荐）
├── skills/                # 依赖的自定义 Skill（可选）
│   └── my-skill.md
└── mcps/                  # 依赖的自定义 MCP 配置（可选）
    └── my-mcp.py
```

### agent.json

`agent.json` 由 Nexent 平台的智能体开发功能导出生成，包含智能体的完整定义：

- 基本信息（名称、描述、ID、作者等）
- 系统提示词
- 参数配置（模型、最大步数等）
- 绑定的 Skill 与 MCP
- 知识库引用
- 示例对话

**不需要手动编写**，在平台中开发完成后导出即可。

### 依赖管理

如果智能体依赖的平台内置 Skill 或 MCP 在目标环境中不存在，导入时会自动提示缺失。如果依赖的是**自定义 Skill 或 MCP**，应按以下方式提供：

- **Skill**：在平台智能体开发中“选择技能”->“构建技能”，上传skill文件。
- **MCP**：在平台智能体开发中“选择工具”->“MCP配置”，按照提示配置需要的MCP工具。

目标用户需先导入这些依赖，再导入智能体本体。

### 开发规范

1. **提示词编写**
   - 清晰定义角色、任务、输出格式
   - 包含输入输出示例
   - 对边界情况做说明

2. **版本管理**
   - 每次功能变更更新版本号并在 CHANGELOG 中注明

3. **测试**
   - 在 Nexent 平台上充分测试后再提交

## 提交智能体到本仓库

### 提交流程

1. **Fork** 本仓库
2. 按目录结构放置智能体文件：
   ```
   AgentsHub/
   ├── 通用智能体/
   │   ├── 研究与分析/
   │   │   └── my-research-agent/
   │   ├──开发与编程/
   │   │   └── my-dev-agent/
   │   └── ...
   └── 行业智能体/
       ├── 医疗/
       ├──金融/
       └── 政务/
   ```
3. 确保 `agent.json` 完整、目录结构正确
4. 如有自定义依赖，确保 `skills/` 或 `mcps/` 目录齐全
5. 提交 Pull Request，PR 标题格式：`[new agent] <display_name>`
6. 维护者审核通过后合并

### 审核标准

- `agent.json` 格式正确、内容完整
- 提示词清晰、无敏感内容
- 已通过 Nexent 平台的基本功能测试
- 不侵犯第三方知识产权
- 不包含恶意代码或数据窃取逻辑
